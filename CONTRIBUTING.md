# Contributing Guidelines

Thank you for your interest in contributing to projects within the LUC-FMitF GitHub organization.

We welcome contributions from students, faculty, collaborators, and the broader research community.

---

## Before You Start

Please:

- Read the README of the specific repository.
- Review open issues and discussions.
- Ensure your contribution aligns with the research goals of the project.
- Follow the Code of Conduct.

If you are unsure about a proposed change, open a discussion or issue first.

---

## Types of Contributions

We welcome:

- Bug reports
- Documentation improvements
- Test cases
- Research prototypes
- Code enhancements
- Performance improvements
- Formal proofs or verification artifacts
- Dataset improvements (when applicable)

---

## Reporting Issues

When submitting an issue:

- Clearly describe the problem.
- Include steps to reproduce (if applicable).
- Provide relevant logs, screenshots, or examples.
- Suggest potential improvements if possible.

---

## Pull Request Process

1. Fork the repository (if external contributor).
2. Create a feature branch with a clear name.
3. Make focused, well-documented commits.
4. Ensure code builds and passes tests (if applicable).
5. Submit a pull request with a clear description of:
   - The purpose of the change
   - Relevant background context
   - Any related issues

Pull requests may undergo review by maintainers before being merged.

---

## Coding Standards

Contributors should:

- Write clear, readable, and well-documented code.
- Follow existing project style conventions.
- Include comments where appropriate.
- Add tests when feasible.
- Avoid unnecessary dependencies.

Research code should prioritize clarity and reproducibility.

---

## Documentation

All significant features or changes should include:

- Updated README sections (if applicable)
- Usage examples
- Relevant citations for research-based implementations

---

## Licensing

By contributing, you agree that your contributions will be licensed under the license specified in the repository.

Ensure that you:

- Have the right to submit the code.
- Properly attribute third-party material.
- Respect open-source licensing terms.

---

## Academic Contributions

If contributing as part of a course, research assistantship, or thesis:

- Coordinate with your supervising faculty member.
- Clearly indicate authorship where appropriate.
- Follow all university academic integrity policies.

---

## Communication

For major architectural or research-direction changes:

- Open a discussion first.
- Allow time for feedback.
- Seek consensus when possible.

---

## Questions?

If you have questions about contributing, open a discussion in the organization or contact a repository maintainer.

We appreciate your interest in supporting rigorous and reproducible research.
